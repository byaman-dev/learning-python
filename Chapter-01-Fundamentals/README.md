## 🏗️ 1. How Computers Actually Work
Before writing code, I learned that a computer is essentially a very fast machine that follows specific instructions. 

* **Hardware vs. Software:** Hardware is the physical machine (CPU, RAM); Software is the set of instructions that tells the hardware what to do.
* **The Language Gap:** Computers speak in **Binary** (0s and 1s). Since humans find binary hard to read, we use **High-Level Languages** like Python.

## 🐍 2. Why Python? (The Guido van Rossum Legacy)
Python was created in 1991 with a focus on **readability**. 
* **Interpreted:** Python code is executed line-by-line by an **Interpreter**.
* **Dynamic:** It’s flexible and doesn't require complex boilerplate code to start.
* **Versatile:** It’s the "Swiss Army Knife" of programming—used for Web, AI, and Automation.

---

## ⚙️ 3. Compilation vs. Interpretation
A key concept from this chapter is how our code actually runs:

| Feature | Compilation (C++, Java) | Interpretation (Python) |
| :--- | :--- | :--- |
| **Process** | Translates the whole file into machine code first. | Translates and runs code line-by-line. |
| **Error Catching** | Finds all syntax errors before running. | Stops at the first error it hits while running. |
| **Speed** | Generally faster. | Slower, but much easier to develop and test. |
