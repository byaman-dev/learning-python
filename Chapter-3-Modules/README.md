# 📦 Chapter 3: Modules, Pip & Comments

In this chapter, I learned how to extend Python's power using external libraries and how to keep my code professional with comments.

## 1. Modules: The Building Blocks
A **Module** is a file containing code written by others that we can import into our project.
* **Built-in:** `os`, `sys`, `math` (Already in Python).
* **External:** `pyttsx3`, `flask`, `tensorflow` (Needs installation).



## 2. Pip: The Package Manager
`pip` is the "App Store" for Python.
* **Install Command:** `pip install <module_name>`
* **Why it matters for AI Security:** Understanding how to manage dependencies is the first step in protecting the software supply chain from "poisoned" packages.

## 3. Comments: Coding for Humans
I learned that code is read more often than it is written.
* **# Single line:** Quick notes.
* **""" Multi-line """:** Used for documentation (Docstrings).
